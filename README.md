## Overview

Traditional predictive maintenance systems rely solely on internal machine telemetry and often fail to capture external conditions that influence equipment failures. This project develops a contextual predictive maintenance framework by integrating IoT sensor data with external environmental and operational context to improve failure prediction accuracy.

The solution combines telemetry-based feature engineering, contextual data fusion, imbalanced learning techniques, and gradient boosting models to predict machine failures before they occur.

---

## Business Problem

Industrial maintenance is traditionally reactive, resulting in:

* Unexpected equipment breakdowns
* Production downtime
* Increased maintenance costs
* Reduced operational efficiency

Machine failures are rarely caused by internal sensor conditions alone. External factors such as environmental conditions, production demand, and operational load significantly influence failure risk.

This project aims to transition maintenance operations from a reactive approach to a proactive, data-driven strategy.

---

## Objectives

* Build a contextual data fusion framework combining IoT telemetry and external signals.
* Predict machine failures using machine learning.
* Handle severe class imbalance using SMOTE.
* Evaluate robustness under noisy sensor conditions.
* Optimize decision thresholds for deployment scenarios.
* Achieve a Macro F1 Score greater than 0.85.

## Dataset

### AI4I Predictive Maintenance Dataset

The dataset contains telemetry collected from industrial machines including:

| Feature             | Description                     |
| ------------------- | ------------------------------- |
| Air Temperature     | Ambient machine temperature     |
| Process Temperature | Internal process temperature    |
| Rotational Speed    | Machine rotational speed (RPM)  |
| Torque              | Torque generated by the machine |
| Tool Wear           | Tool usage duration             |
| Machine Failure     | Target variable                 |

### Dataset Statistics

* Original Records: 10,000
* Final Records After Feature Engineering: 9,991
* Failure Rate: 3.39%

## Methodology

### 1. Telemetry Feature Engineering

Generated advanced time-series features including:

#### Rolling Statistics

* Rolling Mean
* Rolling Standard Deviation
* Rolling Variance

#### Lag Features

* Lag 1
* Lag 2

#### Trend Features

* Rate of Change
* Temperature Difference

---

### 2. Contextual Data Fusion

Additional contextual signals were incorporated:

| Feature                | Description             |
| ---------------------- | ----------------------- |
| Ambient Humidity       | Environmental condition |
| Energy Load Index      | Operational load        |
| Production Demand      | Production intensity    |
| Days Since Maintenance | Maintenance history     |
| Shift                  | Operating shift         |

---

### 3. Interaction Features

Created domain-inspired interaction variables:

* Torque × Load
* Tool Wear × Demand
* Temperature × Humidity

---

### 4. Imbalanced Learning

Machine failures represented only 3.39% of observations.

To prevent data leakage:

* Stratified 5-Fold Cross Validation was used.
* SMOTE was applied **only within training folds**.
* Validation folds remained untouched.

---

### 5. Modeling

Model:

* LightGBM Classifier

Configuration:

```python
n_estimators = 300
learning_rate = 0.05
max_depth = 6
random_state = 42
```
